Questions on data fields [Table 2]:

  1. Device_number

    1. I wanted to confirm: does every gateway have a unique device identifier?
    2. How many devices are shared between control sets?

  2. end_time

    1. What are the expected date ranges for each control set? Do they correspond to the table above [Table 1]?
      • control 1, 2, 3: Oct
      • control 4, 250-test: Oct, Nov, Dec
      • control 5,6,7: Nov (most), Dec (only some)
      • control 8: Dec
    2. Can you estimate how off this synchronized time stamp is (under 30 seconds?)

  3. cmts_inet

    1. Converting these to IP addresses, I notices the pool of different IP addresses is very low. I think I only see 25 distinct IPs in the whole 250-test set.
    2. Why are the IP addresses shared between many devices across sets?
    3. Were both control and test sets chosen only from Salt Lake City, Utah?

  4. service_class_name

    1. What is the relevance of this field? Does it in any way relate to the application generating the data on the users' home network? Or is it internal to a Comcast application?
    2. Does xwifi_up actually mean data on wifi? Or is it the name of a Comcast data counter application I shouldn\'t be concerned with?
    3. extrm_5 has the maximum share of data throughout the 250-test set: what is it?
    4. I am currently disregarding this field and only looking at the sum of octets_passed in each end_time slot for each Device_number. I want to confirm if that is relevant to Comcast (or if I should be choosing some other aggregates).

  5. octets_passed

    1. What is the access link bandwidth of each control set: 105 Mbps?
    2. How hard is this limit, i.e., any idea how much extra Comcast lets a user push through in a 15 min time slot? I saw some transfers as high as 150 Mbps average data rate or so...
